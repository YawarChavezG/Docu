## Contexto general
Estamos iniciando la rama R3 (`r3/workflow-revision-aprobacion`). Antes de implementar las nuevas tablas del workflow, hay que terminar lo que falta del wizard de R2 (Fase 0).

## Reglas para todas las sesiones R3
- Rama de trabajo: `r3/workflow-revision-aprobacion` (crearla desde `r2/wizard-y-version-editable`)
- Cada sesión = 1 fase (no mezclar fases en una misma sesión)
- Cada cambio se valida con:
  - Tests pytest: `cd backend && .venv\Scripts\python -m pytest tests/`
  - Chrome MCP cuando aplique cambios de UI
  - `git diff` antes de commitear
- Al final de cada sesión:
  - Marcar los ítems completados en el checklist
  - Actualizar docs (BITACORA, ESTADO, RADIOGRAFIA)
  - Commit atómico con conventional commit
  - Avisar al usuario que está listo para la siguiente fase
